# 支持

Docusaurus 拥有一个由数千名开发者组成的社区。

在这个页面上，我们列出了一些与 docusaurus 相关的社区，你可以加入其中;有关其他在线和面对面学习材料，请参阅本节的其他页面。

在加入 Docusaurus 社区之前，请阅读我们的[行为准则](https://engineering.fb.com/codeofconduct/)。我们采用了[贡献者公约](https://www.contributor-covenant.org/)，我们希望所有社区成员都遵守其中的指导方针。

## Stack Overflow {#stack-overflow}

Stack Overflow 是一个很受欢迎的论坛，在这里您可以询问代码级别的问题，或者遇到特定的错误。阅读带有 docusaurus 标签的[现有问题](https://stackoverflow.com/questions/tagged/docusaurus)或[问你自己的问题](https://stackoverflow.com/questions/ask?tags=docusaurus)!

## 论坛 {#discussion-forums}

有许多在线论坛可以讨论最佳实践和应用程序架构以及 Docusaurus 的未来。如果您有一个可回答的代码级问题，Stack Overflow 通常是更合适的选择。

- [Docusaurus 在线聊天](https://discord.gg/docusaurus)
  - [#help-and-questions](https://discord.gg/fwbcrQ3dHR) for user help
  - [#contributors](https://discord.gg/6g6ASPA) for contributing help
- [Reddit 的 Docusaurus 社区](https://www.reddit.com/r/docusaurus/)

## 特性请求 {#feature-requests}

对于新的功能请求，你可以在我们的[功能请求板(Canny)](/feature-requests)上创建一个帖子，这是一个方便的工具，可以绘制路线图，并允许通过投票进行排序，这给了核心团队一个更好的指标，哪些功能是高需求的，相比之下，GitHub 问题更难分类。避免为新功能(特别是大型功能)发出 Pull Request，因为有人可能已经在开发它，或者将成为我们路线图的一部分。先跟我们谈谈!

## 新闻 {#news}

有关 Docusaurus 的最新消息，请[关注 Twitter 上的 **@docusaurus**](https://twitter.com/docusaurus)和这个网站上的[Docusaurus 官方博客](/blog)。
