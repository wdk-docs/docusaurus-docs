# 接下来是什么?

恭喜你!现在你已经了解了 Docusaurus 的大部分核心特征。你有:

- [使用 pages 插件](./creating-pages.mdx)创建一个独立的 React / Markdown 页面
- [使用 docs 插件](./docs/docs-introduction.mdx)创建文档页面。这包括[配置侧边栏](./docs/sidebar/index.mdx)，甚至还有[版本控制](./docs/versioning.mdx)。
- [使用博客插件](../blog.mdx)创建一个功能齐全的博客
- 尝试了一系列 [Markdown 特性](./markdown-features/markdown-features-intro.mdx)，它们对所有内容插件都很有用
- [使用的样式表](../styling-layout.mdx)或[搅拌](../swizzling.mdx)来定制您网站的外观
- [包括图像和其他资产](../static-assets.mdx)在您的页面
- [添加搜索](../search.mdx)到您的网站
- 了解[浏览器支持](../browser-support.mdx)和[SEO](../seo.mdx)是如何通过标准 Docusaurus api 完成的
- 学习了如何安装和配置[单个插件](../using-plugins.mdx)
- [部署](../deployment.mdx)您的网站到一个内容主机
- [国际化](../i18n/i18n-tutorial.mdx)你的网站包含多种语言

此时，您可能已经有了一个很大的`docusaurus.config.js`😄 但是，您还没有编写太多代码!大多数功能都是通过调用封装的 Docusaurus api 实现的。当你继续你的旅程时，你可以走三条路:

- 学习更多先进的 Docusaurus 概念。这将帮助您更深入地了解 api 的作用。
- 阅读[所有现有 api](../docusaurus-core.mdx)。其中许多都没有在指南中提到!
- 学习[开发插件](../api/plugin-methods/README.mdx)来扩展您网站的功能。
